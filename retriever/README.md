# Module B — Data & Retrieval Pipeline

**Role**: Data & Retrieval Engineer  
**Layer**: 检索层

---

## Overview

This module builds the retrieval backbone of the RAG pipeline. It downloads and processes PubMed medical abstracts, chunks them into sentence-level units, and provides a unified hybrid retrieval interface combining BM25 sparse retrieval and Dense vector search.

## Pipeline Steps

### Step 1 — Download PubMed Data
Searches PubMed for `"diabetes"` and downloads up to 10,000 abstracts via the E-utilities API. Results are saved as raw XML batches.

- Output: `raw_abstracts.json`

### Step 2 — Parse XML
Parses raw XML from Step 1 and extracts structured fields. Articles without abstracts are skipped.

- Output: `abstracts.json`
- Fields: `pmid`, `year`, `abstract`, `mesh`

### Step 3 — Chunk Abstracts
Splits each abstract into sentence-level chunks. Each chunk retains source metadata including PMID, year, MeSH terms, and position. Fragments under 20 characters are discarded.

- Output: `chunks.json`
- Fields: `pmid`, `year`, `mesh`, `chunk_id`, `position`, `text`

### Step 4 — Build Vector Index
Encodes all chunks using `all-MiniLM-L6-v2` and builds a FAISS flat inner product index. Vectors are L2-normalized for cosine similarity. Only needs to be run once.

- Output: `embeddings.npy`, `faiss.index`

### Step 5 — Test Retrieval
Loads all indexes and runs a test query through the hybrid retrieval interface. Outputs top-k results with BM25, Dense, and fusion scores.

## Retrieval Interface

Other modules can import `retrieve()` directly:

```python
from pipeline import retrieve

results = retrieve(query="insulin resistance in diabetic patients", top_k=5)
```

### Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `query` | `str` | — | Natural language search query |
| `top_k` | `int` | `5` | Number of results to return |

### Return Schema

Each result is a `dict` with the following fields:

| Field | Type | Description |
|-------|------|-------------|
| `score` | `float` | Hybrid fusion score (0–1) |
| `bm25` | `float` | Normalized BM25 score |
| `dense` | `float` | Cosine similarity score |
| `pmid` | `str` | PubMed article ID |
| `year` | `str` | Publication year |
| `mesh` | `list[str]` | MeSH terms (may be empty for recent articles) |
| `chunk_id` | `str` | Unique chunk ID: `{pmid}_{position}` |
| `position` | `int` | Sentence index within the abstract |
| `text` | `str` | Chunk text content |

---

## Configuration

Key parameters can be adjusted at the top of `pipeline.py`:

| Variable | Default | Description |
|----------|---------|-------------|
| `SEARCH_QUERY` | `"diabetes"` | PubMed search keyword |
| `MAX_RESULTS` | `10000` | Maximum PMIDs to fetch |
| `BATCH_SIZE_API` | `200` | Articles per API request |
| `BATCH_SIZE_ENC` | `256` | Chunks per encoding batch |
| `EMBED_MODEL` | `all-MiniLM-L6-v2` | Sentence embedding model |
| `BM25_ALPHA` | `0.5` | BM25 weight in hybrid fusion |

## Integration Notes

### For Module C (Generator)

```python
from pipeline import retrieve

def build_context(query, top_k=5):
    results = retrieve(query, top_k=top_k)
    context = "\n".join([
        f"[PMID {r['pmid']}] {r['text']}"
        for r in results
    ])
    return context, results
```

### For Module A (Integration)

- Entry point: `pipeline.py`
- Required files at runtime: `chunks.json`, `faiss.index`
- Indexes load once at first `retrieve()` call (~2–3 seconds on GPU)
- Step 4 only needs to run once to generate index files
